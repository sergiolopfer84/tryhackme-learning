## Summary
This room focused on practical Linux fundamentals relevant to system administration and cybersecurity.

## Key concepts learned
- Remote access to Linux systems using SSH and understanding authentication context.
- Effective use of command flags and manual pages to interpret and adapt command output.
- File system navigation and management, including the distinction between user identity and directory location.
- User switching (`su`, `su -`, `sudo`) and the security implications of environment inheritance.
- Overview of critical Linux directories such as `/etc`, `/var`, `/root`, and `/tmp` and their relevance in security contexts.

## Security relevance
Understanding user context, file locations, and system configuration is essential for enumeration, privilege escalation, and incident response in Linux environments.
